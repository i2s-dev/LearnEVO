This update check was first introduced in version 2000.2. If it has never been run before, it is very likely that a number of minor
record length (Type 3) errors will be detected, which have long existed and do not affect function but should still be corrected
using the steps listed below. There is no need to call DBA Technical Support unless the steps indicated do not resolve the
discrepancies.
The numbered instructions below are to be used to for fixing the above error(s). Look in the Action Type column in the report above
to see which numbers apply. For each Action Type number listed, you will find a corresponding numbered instruction below. Please
preform the appropriate action(s) to correct the error(s).
1. Dictionary and UPD Files do not match. Manually re-run UPDATE program from the windows taskbar as follows:
Select Start > Run, and type in X:\DBAMFG\TP5WDBA UPDATE (where X:\DBAMFG is your path to the DBAMFG directory). Click OK.
2. You have an extra file or files in your data dictionary that are not contained in the standard DBA dictionary. This may be from
a customization. No immediate action necessary but be sure any customizations have been updated as needed to be compatible with
other data file changes. If you do not have any customizations, please call DBA Technical support.
3. You have a file or files that have a record length that varies from the data dictionary. You need to re-run the update for those
files only. For any files that have an Action Type of "3" in the reportabove, note their names as shown in the Layout column. Be
sure you have backup copies of these files, and then do the following:
 a. In the DBA System Manager module, select UT-A. Enter the program name UPDFILE, then press the Enter key. You will be prompted
 to select the data structures to be updated. Highlight each file name that matches a file name shown in the Layout column of the
 report above. Use the TAG/Untag button in the lower left corner to select it for updating. (When a file has been selected, a check
 mark appears to the left of it.) When you have finished selecting all of the files to be updated, click the Process button.
 b. Once processing is complete, select UT-A again. Enter the program name DDCHECK, then press the Enter key. This action re-run
 the dictionary check routine to confirm all the differences have been corrected. If the routine encounters any errors, you will
 have the opportunity to view them as well as print a report.
4. You have one or more missing files as determined by comparing the dictionary list to the actual data file directory.
Do the following:
 a. In the DBA System Manager module, select UT-A. Enter the program name NEWINIT, then press the Enter key. This routine will
 create any missing files.
 b. Once processing is complete, select UT-A again. Enter the program name DDCHECK, then press the Enter key. This action re-runs
 the dictionary check routine to confirm that all the differences have been corrected. If the routine encounters any errors, you
 will have the opportunity to view them as well as print a report.
