What does the Master Level reconciliation do?F
---------------------------------------------F
NOTE: The information in this section is provided for technical reference F

*  Deletes any blank records in the BKICMSTR, MTICMSTR, BKICLOC, CLASMSTR,F
   CLASS, INVTXN, and DBAFIFO files.F
*  Checks BKICLOCM (Location Master file).  If it has no records, adds oneF
   for the default (Inventory Defaults) location.  If there are records, but none for   F
   the default (Inventory Defaults) location, adds one for that location.  Deletes any  F
   duplicate BKICLOCM records.F
*  Checks CLASMSTR (Class Master file).  If it has no records, creates    F
   one for Class "DFLT".  Deletes any duplicate CLASMSTR records.         F
*  Deletes any duplicate records found in CLASS (Class/Location file).    F
   Any Classes in CLASS that do not exist in CLASMSTR are added to        F
   CLASMSTR.  Any Locations in CLASS that do not exist in BKICLOCM are    F
   added to BKICLOCM.F
*  Recovers any "stranded"BKICMSTR/MTICMSTR and BKICLOC records (records without   F
   a corresponding BKICMSTR/MTICMSTR record for that Part.)F
The following are done for every Part within the range specified by user: F
*  Checks important BKICMSTR (Inventory Master file) fields.  If Type is  F
   not valid (NRMFABLTKO), checks if Type in the MTICMSTR file (Auxiliary F
   Inventory Master) for this part is valid.  If so, changes Type in      F
   BKICMSTR to that value.  If not, sets BKICMSTR Type to 'R'.  If the    F
   Stock U/M in BKICMSTR is blank, sets that to be the same as MTICMSTR   F
   Selling U/M (if that is also blank, then sets Stock U/M to 'EA').      F
*  If the BKICMSTR Class is not valid (not found in CLASMSTR file), tries F
   to use the MTICMSTR Class.  If that is also not valid, creates a new   F
   record in CLASMSTR for the BKICMSTR Class.  If the BKICMSTR Class is   F
   blank, sets BKICMSTR Class to be the first Class found in CLASMSTR     F
   instead of creating a blank CLASMSTR record.F
*  If the MTICMSTR record for a given Item No. is missing, creates it.    F
   Uses Class, Type, Description, from BKICMSTR.  Sets Purchase U/M and   F
   Selling U/M to be same as BKICMSTR Stock U/M.  Sets Active Flag to Y,  F
   Purchase Conversion to 1.00.  Sets the MRP Flag and the Expedite/Delay F
   Buffers to the MD-I defaults.F
*  If Class, Type, or Desc in MTICMSTR are not the same as in BKICMSTR,   F
   changes them in MTICMSTR.  If the MTICMSTR Purchase Conversion Factor  F
   is zero, sets it 1.00.  If the MTICMSTR Purchase U/M or Selling U/M areF
   blamk, makes them the same as Stock U/M from BKICMSTR.  If the MRP FlagF
   in MTICMSTR is blank, sets it to the MD-I default setting.F
*  Deletes any duplicate BKICMSTR and MTICMSTR records the Part may have. F
*  Checks Location records (BKICLOC) for the Part.  If the PartF
   does not have any BKICLOC records, creates one using the Inventory F
   Defaults default location code.F
*  Makes sure that there is a matching BKICLOCM recordF
   for each BKICLOC location for that Part.  If not, BKICLOC getsF
   Reset to the Default Location.F
*  Consolidates then Deletes any duplicate BKICLOC records. F
*  If the Part is Type N, K, L, T, B, or O and Units on Hand (UOH), Total F
   Value (TOTVL), or Average Cost (AVGC) in BKICMSTR are not zero, resets F
   these numbers to zero.  Then, if any of the BKICLOC records for the    F
   Part have a non-zero UOH, resets UOH for that location to zero.F
*  Rebuilds unit counts (except UOH) for each location (BKICLOC record)   F
   for Part from open orders within the system:  Units on SO and BackorderF
   from open SO's, units on PO and in QC from open PO's, and units on WO, F

   the sum of the BKICLOC unit counts for all locations for the Part.     F
   If not, the BKICMSTR/MTICMSTR totals are changed.  Finally,            F

What does the Method A Transaction Level reconciliation do?F
-----------------------------------------------------------F
Use Method A to make the INVTXN (IN-E) transactions match the master      F
inventory files.  These are the steps performed by Method A:              F
*  Deletes any INVTXN records whose transaction type is not ASPJWIQTGR.   F
   Also deletes any INVTXN records where a corresponding record for the   F
   part does not exist in BKICMSTR.F
*  For Type RFAM parts:F
   1. If there are no INVTXN records at all for the part, creates one for F
      each location (BKICLOC record) where UOH for the part is not zero.  F
   2. Adds up the INVTXN records to get a UOH total by location for part. F
   3. If any of the location totals are non-zero, but there is no match-  F
      ing BKICLOC record for that location, Changes the INVTXN records    F
      to the Default location.F
   4. If UOH total for location according to INVTXN does not match UOH of F
      the BKICLOC record, creates an adjusting INVTXN entry.              F
What does the Method B Transaction Level reconciliation do?F
-----------------------------------------------------------F
Use Method B to make the master inventory files match the INVTXN (IN-E)   F
transactions.  These are the steps performed by Method B:                 F
*  Deletes any INVTXN records whose transaction type is not ASPJWIQT.     F
*  If a part has transactions but no BKICMSTR record, lets user choose to F
   create the BKICMSTR record (on-the-fly, via IN-B), or delete the trxns.F
*  Changes the BKICLOC UOH total for each location to match the trxns UOH F
   total for every location.  Creates the BKICLOC records if needed.      F
*  Changes UOH in BKICMSTR to match changes made to BKICLOC.              F
*  Makes a second pass to zero out UOH (in BKICMSTR and all BKICLOC       F
   locations) for parts that do not have any transactions.I
