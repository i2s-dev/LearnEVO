Please Read Before Proceeding                                             F

Work Order File MaintenanceF
---------------------------F
This program enables you to rapidly clean up any inconsistencies F
in your Work Order and other manufacturing data.F
The WO File Maintenance ReportF
------------------------------F
A report will be generated detailing any changes that may have been F
made (you will be able to choose whether to print or display theF
report after the maintenance process is complete).  The far leftF
column will display an "*" for items that this program was unable to resolve F
or that need more attention on your part.  This column will also display  F
a "$" sign for items that may affect General Ledger amounts.F
You can also choose to run the maintenance in 'report-only' mode.      F
In this case, no changes will be made to your data.  Only a report        F
detailing anticipated changes will be generated, so that you may preview  F
what will be done without actually changing anything.F
Please bear in mind, however, that when you subsequently run this         F
process 'for real', the actual changes made may not exactly match the   F
initial report.  This is because changes made later in the process are    F
dependent upon changes made earlier in the process.  Thus problems fixed  F
in early steps may mean that changes the report has indicated are needed  F
in later steps are no longer necessary.  Also, some of the anticipated    F
adjustments may look strange because they are using numbers that have not F
yet really been fixed in the earlier steps.F
Another factor contributing to differences between anticipated and actual F
changes could be the actions of other users posting transactionsF
between the time you ran the report and the time you actually made the    F
maintenance changes.                                                   F
Suggested StrategyF
------------------F
If you wish to back up your data before running this process,  F
the following files may be affected: OUTPROC, WOLABOR, WOMAT, WOROUT, F
WOBOM, WODATE, WORKORD, WOEXCHG, WORECV, BKDCLAB, and WOBOMREM. F
This is an extensive process and can take some time to run, depending on  F
how much data needs to be analyzed.  In addition it is interactive, so youF
may be prompted for information during the run if this program encounters F
a situation that it cannot resolve by itself.  Such cases will be         F
indicated on the "report-only" report as "Program will prompt you".F
If you choose NOT to run in report-only mode (so that data will get       F
changed), the program will not run if any other users are currently       F
working in this company.  This is done to ensure that the process does notF
encounter a transaction that has not yet completely posted to all files,  F
thus resulting in an unwarranted adjustment being made.  If you run this  F
process in report-only mode (where no data will get changed), you can do  F
so while other users are working with this company.  However, the report  F
may still indicate unnecessary adjustments if incomplete transactions are F
encountered.  In this case you should ensure that other users will be     F
doing tasks that do not affect work order related files.F
A suggested strategy might be as follows:                  F
1. RUN IN REPORT-ONLY MODE FIRSTF
     Run the maintenance process in report-only mode.  This will   F
     give you a feel for the extent of necessary changes and you can      F
     decide whether or not to proceed with them.                          F
2. RUN THE MAINTENANCE PROCESS 'FOR REAL'F
     After reviewing the report, go ahead and run the maintenance F
     process 'for real'.  You'll get a second report indicating the     F
     actual changes made.F
What does Work Order File Maintenance do?F
-----------------------------------------F
NOTE: The information in this section is provided for technical reference F
only.  You do not need to know it to effectively use this program and can F
skip this section if desired.F
*  Deletes any blank records (ones that have a work order number of zero) F
   in the following files: OUTPROC, WOLABOR, WOMAT, WOROUT, WOBOM, WODATE,F
   WORKORD, WOEXCHG, WORECV, BKDCLAB, WOBOMREM.                           F
*  Deletes any orphan records (ones that do not have a parent WORKORD     F
   record) in the following files: OUTPROC, WOLABOR, WOMAT, WOROUT, WOBOM,F
   WODATE, WOEXCHG, WORECV, BKDCLAB, WOBOMREM.F
*  Identifies any duplicate work order numbers in WORKORD.                 F
*  Identifies any records having an invalid work order suffix (a suffix    F
   that is not in the 1-999 range) in the following files:                 F
   WORKORD, OUTPROC, WOLABOR, WOMAT, WOROUT, WOBOM, WODATE, WOEXCHG,       F
   WORECV, BKDCLAB, WOBOMREM.                                              F
*  Identifies any records in OUTPROC, WOLABOR, WOMAT, WODATE, WOEXCHG,     F
   WORECV, and BKDCLAB that have a transaction date that is earlier        F
   that the actual start date of the WO.                                   F
   Records such as these need to be looked at as they may indicate a work  F
   order number that was incompletely purged, leaving some records for thatF
   WO in the subsidiary files.  If the WO number has gotten reused, those  F
   subsidiary records will have "attached" themselves to the new WO, thus  F
   distorting costs of the new WO.F
